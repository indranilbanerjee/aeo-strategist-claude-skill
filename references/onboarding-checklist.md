# New Client Onboarding Checklist — AEO Retainer
## INT TechShu Internal | Use with /onboard mode

---

## Purpose
Every new AEO retainer client must go through this intake before any deliverable is produced. Skipping this creates deliverables that are wrong in ways that take months to unwind.

---

## Section 1: Business Foundation

**1.1 Company identity**
- [ ] Legal company name (as registered with MCA/CIN or equivalent)
- [ ] Brand name(s) used publicly (may differ from legal name)
- [ ] Website URL (all domains and subdomains)
- [ ] Year founded
- [ ] Ownership structure (public / private / subsidiary — matters for Wikidata + entity trust)

**1.2 Business model**
- [ ] Revenue model: Product / Service / SaaS / Marketplace / Hybrid
- [ ] B2B / B2C / B2B2C
- [ ] Primary buyer persona (job title, context, level of sophistication)
- [ ] Geographic markets served (primary + secondary)

**1.3 Competitive positioning**
- [ ] 3 primary competitors (by name)
- [ ] Competitive advantage claim (what the brand says about itself)
- [ ] Market position: New entrant / Challenger / Established / Leader
- [ ] Brand awareness level: Unknown in category / Known / Category-defining

---

## Section 2: Regulatory & Compliance Status

**2.1 Industry registrations (collect ALL applicable)**
- [ ] Company registration number (CIN/ROC — India; equivalent for other markets)
- [ ] Industry-specific licence numbers (SEBI, RBI, IRDAI, CDSCO, RERA, BIS, FSSAI, etc.)
- [ ] Any pending regulatory actions or disputes (disclosure — needed for risk assessment)
- [ ] Advertising/content restrictions known to client (existing legal guidance)

**2.2 Compliance programme status**
- [ ] Does client have a medico-legal/compliance review team? (Pharma/BFSI/Healthcare)
- [ ] What is the review SLA for content? (Critical for retainer planning)
- [ ] Any prior regulatory warnings for advertising/marketing?

---

## Section 3: Current Digital Footprint Audit

**3.1 Website technical baseline**
- [ ] CMS (WordPress, custom, Webflow, Shopify, etc.)
- [ ] Is the site React/Vue/Angular SPA? (JS rendering risk — CRITICAL)
- [ ] Does it use SSR or SSG? (Next.js, Nuxt, Gatsby, etc.)
- [ ] Is there a sitemap? (URL: sitemap.xml)
- [ ] robots.txt — current configuration (copy full content)
- [ ] Existing schema markup? (Validate via schema.org validator or Google Rich Results Test)
- [ ] Page speed baseline (Core Web Vitals from Search Console)

**3.2 Search Console baseline**
- [ ] Google Search Console verified? (Share access or provide screenshot)
- [ ] Bing Webmaster Tools verified? (Critical for ChatGPT — flag if not)
- [ ] Google Merchant Center? (Retail/E-commerce)
- [ ] Current top performing pages by organic traffic (top 10)

**3.3 Entity presence**
- [ ] Wikipedia page? (URL if yes)
- [ ] Wikidata entry? (QID if yes)
- [ ] Google Knowledge Panel? (Show screenshot)
- [ ] LinkedIn company page URL
- [ ] Crunchbase listing? (For tech/startup clients)

**3.4 Directory presence (industry-specific)**
Complete a directory audit for the relevant vertical using the industry file. Mark each:
- Listed and complete ✅
- Listed but incomplete ⚠️
- Not listed ❌
- Not applicable (N/A)

---

## Section 4: Existing Content Infrastructure

**4.1 Content production**
- [ ] Blog / content hub URL
- [ ] Average publishing frequency (articles/month)
- [ ] Content team composition (in-house writers, agency, freelancers)
- [ ] Review process for content (who approves before publishing)
- [ ] Existing content guidelines or brand voice document

**4.2 Content audit (top pages)**
- [ ] Top 10 pages by organic traffic
- [ ] Top 5 pages by conversions
- [ ] Any existing FAQ pages?
- [ ] Any existing schema-marked pages?

---

## Section 5: PR & Media Relations

**5.1 Media footprint**
- [ ] Last 12 months media coverage (share press clippings or media list)
- [ ] PR agency or in-house PR?
- [ ] Existing media relationships (which journalists/publications)
- [ ] Any AI-licensed publication coverage in last 12 months? (Bloomberg, Reuters, FT, ET, etc.)

**5.2 Awards and recognition**
- [ ] Industry awards won (last 3 years)
- [ ] Rankings achieved (Gartner, G2, Clutch, etc.)
- [ ] Government recognition or grants received

---

## Section 6: Current AI Visibility Baseline

**6.1 Client's own knowledge**
- [ ] Has client tested what AI says about their brand? (Ask them to test now if not)
- [ ] Any known inaccuracies in AI responses about them?
- [ ] Do they appear in any AI answers? For which queries?

**6.2 Initial manual test (conduct before first call)**
Before the onboarding call, run these tests manually:
1. "[Brand name]" — what does Gemini/ChatGPT/Perplexity say without any additional context?
2. "[Brand's primary category] best [market]" — do they appear?
3. "[Brand name] vs [top competitor]" — how are they described?
4. "[Brand name] reviews" — what does AI say?

Document results. Use as conversation starter in kickoff.

---

## Section 7: Goals & Success Metrics

**7.1 Business objectives**
- [ ] Primary business goal (lead gen / brand awareness / e-commerce sales / investor credibility)
- [ ] Why AEO specifically? (What triggered this engagement)
- [ ] Timeline expectations (are they realistic about 12–18 month parametric timelines?)

**7.2 Reporting preferences**
- [ ] Reporting frequency (monthly minimum recommended)
- [ ] Reporting audience (CMO, CEO, marketing team, board?)
- [ ] Preferred format (deck / doc / live meeting / async)
- [ ] KPIs agreed (AI SOV % / citation rate / brand search volume / referral traffic)

---

## Onboarding Deliverables (First 30 Days)

After completing this intake, produce in this order:
1. Client brief (1-page summary of answers above)
2. `/audit` — full AEO readiness assessment
3. SOV baseline (manual query test — Layer 1 from tracking-protocols.md)
4. Quick wins list (3 actions completable this month)
5. `/strategy` — full AEO strategy document
6. `/roadmap` — 20-week implementation plan
7. `/track` — tracking setup protocol for this client

First client-facing deliverable: the audit + quick wins list.
